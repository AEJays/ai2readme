## Project Overview

This project helps you integrate project content and generate markdown, aiming to combine all scenarios that are convenient for programmers, such as changelog / readme / commit, allowing programmers to focus more on developing code.

### How Project Parts Are Obtained

1. **GitHub Repository Acquisition**: Mainly implemented through `git remote -v`, and the GitHub logos above require you to have set the `origin` git remote. Your repository must also be an open-source repository.

2. **Different Module Analysis for README**: Uses different prompts for AI analysis. The modules currently in use have undergone extensive testing, and you can replace them with prompts more suitable for your own modules. You can also add your own modules based on your modules.

3. In `ai2readme generate`, the main file is handled by default using the `main` field from `package.json`. Other types of code may not have a `package.json`, so I made it configurable for JavaScript projects to be more comfortable to use. The name is also default to the `name` field from `package.json` unless your project doesn't have one, in which case it will get the folder name. The `--config` mode is mostly for backend use, but frontend can also use it as long as you specify the configuration file (it will read your configuration file for the large model when needed).

4. All development here is based on `glm-4.6v-flash` generation. If you replace it or use ollama local models, please test it yourself.

## Usage

### How to Run

#### Prepare Environment
1. **Configure Environment Variables (Optional, only for Zhipu AI)**  
   Copy the `.env.example` file and rename it to `.env`, then add the API key:
   The content of .env.example is as follows:
   ```env
   # Choose AI backend, optional zhipu or ollama
   AI_PROVIDER=zhipu
   # Zhipu AI's API key (go to https://open.bigmodel.cn/ to get)
   ZHIPU_API_KEY=<Your ZHIPU AI API Key>
   # Model provided by Zhipu AI, currently recommended free models
   ZHIPU_MODEL=glm-4.6v-flash # Currently Zhipu AI's free better model (tested to be usable)
   # Base URL for ollama service, default local port 11434
   OLLAMA_BASE_URL=http://localhost:11434
   # Local model name used by ollama
   OLLAMA_MODEL=<YourModel>
   ```

2. **Run Commands**  
   Use the `ai2readme` command, supporting the following subcommands:
   - `generate`: Generate project README
   - `translate`: Translate the generated README to English
   - `help`: Display help information

#### Command Examples
- **Generate README for the current directory**:
  ```bash
  ai2readme generate
  ```
- **Specify target directory and main file**:
  ```bash
  ai2readme generate ./my-project src/index.js
  ```
- **Translate the generated README (reads Chinese README => English README, only reads ./my-project/README.md)**:
  ```bash
  ai2readme translate ./my-project
  ```

#### Interactive Examples
1. **Display help information**:

## Features

| Feature | Description |
| --- | --- |
| Multi-AI Backend Support | Supports Zhipu AI cloud (zhipu) and Ollama local models as AI service providers |
| Project Structure Automatic Analysis | Automatically scans project directories to generate file structure information |
| Key Code Snippet Extraction | Extracts core code snippets of the project (with limits on file count and single file line count) |
| Automatic README Generation | Automatically generates professional Chinese README documents based on AI analysis of project information |
| README Translation Feature | Supports translating the generated README document to English version |
| Command Line Interaction | Provides generate (generate), translate (translate), help (help) command line operations |
| Flexible Configuration Options | Can customize ignore patterns, code file types, output directory, etc. through configuration files |
| Robust Error Handling | Includes error logging and program exit mechanisms |

## Technology Stack

### Dependency Overview
The following are the Node.js dependencies used by the project, organized by functionality:

- **Tool Libraries**
  - `dotenv (^17.2.3)`: Loads environment variables for easy configuration management
  - `fs-extra (^11.3.3)`: Extends Node.js file system operations to support richer file operations
  - `ignore (^7.0.5)`: Provides file/directory ignore rules, commonly used to exclude specific files in build processes
  - `openai (^6.16.0)`: OpenAI API client, used to call OpenAI services (e.g., model calling, API interaction)
